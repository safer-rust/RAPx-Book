# Chapter 8.3. Safety Property Contracts

RAPx verifies safety through explicit contracts declared in source code. This chapter covers how to write contracts using the available property kinds, and how to inspect resolved contracts with `--debug-contracts`.

Contracts are declared in two ways:

1. **Direct annotation** тАФ `#[rapx::requires(...)]`, `#[rapx::invariant(...)]`, `#[rapx::ensures(...)]` attributes in Rust source ([┬з8.3.2](#832-direct-annotation-syntax)).
2. **JSON database** тАФ pre-written contracts for standard-library functions that cannot be annotated upstream ([┬з8.3.4](#834-json-contracts)).

## 8.3.1 Property Kinds

The semantics of each property kind are defined in [`primitive-sp.md`](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md). An annotated reference with examples for each kind is in [┬з8.3.3](#833-property-reference).

## 8.3.2 Direct Annotation Syntax

Contracts written in source use `register_tool` tool attributes. The crate (or module) must enable the tool:

```rust
#![feature(register_tool)]
#![register_tool(rapx)]
```

### `#[rapx::requires(...)]`

Declares a precondition on an (usually `unsafe`) function. Each attribute holds one property; group with commas or repeat the attribute:

```rust
// Repeated attributes тАж
#[rapx::requires(ValidPtr(ptr, u32, 1))]
#[rapx::requires(Align(ptr, u32))]

// тАж are equivalent to a single grouped attribute
#[rapx::requires(ValidPtr(ptr, u32, 1), Align(ptr, u32))]
```

A complete example:

```rust
#[rapx::requires(ValidPtr(ptr, u32, len))]
#[rapx::requires(Align(ptr, u32))]
#[rapx::requires(InBound(ptr, u32, len))]
#[rapx::requires(ValidNum(index < len))]
pub unsafe fn write_slice(ptr: *mut u32, len: usize, index: usize) {
    // ...
}
```

### `#[rapx::verify]`

Marks a function as a verification entry point for `--mode targeted` ([Chapter 8.1.2](./8.1-overview.md#812-verification-modes)). It takes no arguments and is orthogonal to `#[rapx::requires]`.

### `#[rapx::invariant(...)]`

Declares a struct-level invariant that must hold for every instance at all observable points ([Chapter 8.4.6](./8.4-pipeline.md#846-struct-invariant-verification)):

```rust
#[rapx::invariant(ValidPtr(ptr, T, cap))]
#[rapx::invariant(Align(ptr, T))]
pub struct MyBuf<T> {
    ptr: *mut T,
    len: usize,
    cap: usize,
}
```

### `#[rapx::ensures(...)]`

Declares a postcondition on an **unsafe trait** method тАФ a contract that every `impl` of the trait must satisfy. Whereas `#[rapx::requires]` defines obligations on the *caller*, `#[rapx::ensures]` defines obligations on the *implementor*:

```rust
#![feature(register_tool)]
#![register_tool(rapx)]

unsafe trait Buffer {
    #[rapx::ensures(ValidSlice(return, u8))]
    fn as_bytes(&self) -> &[u8];

    #[rapx::requires(InBound(index, u8, 1))]
    unsafe fn get_unchecked(&self, index: usize) -> u8;
}
```

- The special placeholder `return` refers to the method's return value.
- Postconditions on trait methods can reference `self` fields and method parameters, same as `requires`.
- `#[rapx::ensures]` **only** applies to `unsafe trait` methods. Regular functions do not use (or need) this attribute тАФ their post-effects are inferred from the standard-library call summary database.
- Trait implementation verification is currently **deferred** (logged during `--prepare-targets` but not yet proven by the SMT engine).
- Trait methods that take `unsafe fn` should use `#[rapx::requires]` for their preconditions and may additionally use `#[rapx::ensures]` for postconditions visible to clients.

Contract properties inside `#[rapx::ensures(...)]` use the same `PropertyKind` catalog and argument grammar as `#[rapx::requires(...)]` ([┬з8.3.1](#831-property-kinds)).

### `any(...)` Combinator

The `any(...)` combinator represents logical OR between disjuncts, where commas inside each parenthesized disjunct mean logical AND:

```text
any((Null(p)), (P1(p, ...), P2(p, ...), ...))
```

The currently supported shape is a **null guard**: exactly two disjuncts, one being `Null(p)` alone, the other a conjunction of properties over the same place `p`. The semantics: the conjunct properties hold whenever `p` is non-null, and the whole contract is satisfied when `p` is null. This works in both `#[rapx::requires]` and `#[rapx::invariant]` positions:

- **In `requires`**: The SMT solver encodes `(p == 0) тИи (P1 тИз P2)`. If the caller's state satisfies either branch, the contract is proved. This is useful for functions like `ptr::as_ref()` that accept null pointers тАФ the contract declares the pointer may be null, but must be valid and aligned when non-null:

```rust
#[rapx::requires(any(Null(self), (ValidPtr(self, T, 1), Align(self, T))))]
pub unsafe fn as_ref<'a, T>(self: *const T) -> Option<&'a T> { /* ... */ }
```

- **In `invariant`**: Used for struct fields that may be null (e.g., linked list tail pointers). Combined with dominance information from null checks in user code (`if ptr.is_null() { return; }`), the verifier determines which branch to prove:

```rust
#[rapx::invariant(any((Null(next)), (ValidPtr(next, Node, 1), Align(next, Node))))]
pub struct Node {
    value: u32,
    next: *mut Node,
}
```

In both cases, `any(Null(p), (P1, P2))` expands to individual `Property` objects for `P1` and `P2`, each tagged with the null-guard place. The SMT checker then asserts `p != 0` before proving each conjunct, which is equivalent to proving `(p == 0) тИи P_i_holds`.

### Contract Kind Metadata

A property may carry a trailing `kind = "<string>"` tag inside the same attribute. Two `ContractKind` values are recognized:

- **`kind = "precond"`** (default): A normal safety precondition. An `Unproved` verdict for this kind contributes to the `UNSOUND` result.
- **`kind = "hazard"`**: An advisory warning about a known unsafety pattern. An `Unproved` verdict for this kind **does not** contribute to the `UNSOUND` count тАФ it is printed as an informational `[hazard]` entry. Set via `kind = "hazard"` inside `#[rapx::requires]`:

```rust
#[rapx::requires(ValidNum(size_of::<T>() * len <= isize::MAX), kind = "hazard")]
pub unsafe fn from_raw_parts<T>(ptr: *const T, len: usize) -> *const [T] {
    // ...
}
```

The `Alias` property is always classified as hazard automatically. Hazard contracts are also used in `std-public-contracts.json` to flag known but currently unprovable constraints.

### Places accepted in annotations

- Parameter names in `requires` / `ensures`: `self`, `ptr`, `index`, `len`.
- Field names in `invariant`: `ptr`, `cap`, `head` тАФ no `self.` prefix.
- Field projections in `requires` (via `self`): `self.0`, `self.ptr`.
- Length sugar: `self.len` is read as `len(self)`.
- Const generics: `N` resolves to the enclosing item's const parameter.
- Return value: `return` refers to the function's return value (only in `#[rapx::ensures]` context).

## 8.3.3 Property Reference

### `ValidPtr(ptr, Ty, count)`

The pointer is valid for reading and writing `count` elements of type `Ty`. This is the fundamental pointer-validity contract.

```rust
#[rapx::requires(ValidPtr(ptr, u32, len))]
```

### `Align(ptr, Ty)`

The pointer satisfies the alignment requirement of type `Ty`.

```rust
#[rapx::requires(Align(ptr, u32))]
```

### `InBound` / `InBounded`

Every accessed byte lies within the bounds of the owning allocation. Several forms are accepted:

| Form | Example |
|------|---------|
| `InBound(slice, index)` | `#[rapx::requires(InBound(self, i))]` |
| `InBound(index_access(slice, index))` | `#[rapx::requires(InBound(index_access(self, i)))]` |
| `InBound(ptr, Ty, count)` | `#[rapx::requires(InBound(ptr, u32, len))]` |
| `InBound(ptr, count)` | `#[rapx::requires(InBound(ptr, len))]` |

The `index_access` form supports all `SliceIndex` types (`usize`, `Range<usize>`, `RangeInclusive<usize>`, etc.) and auto-expands to the correct bounds check.

### `Init(ptr, Ty, count)`

Memory is initialized for `count` elements of `Ty`. Used to prove reads are safe.

```rust
#[rapx::requires(Init(ptr, T, len))]
```

### `NonNull(ptr)`

The pointer is not null.

```rust
#[rapx::requires(NonNull(ptr))]
```

### `Allocated(ptr)` / `Allocated(ptr, Ty, count)`

Memory belongs to a live allocation and the pointer is within bounds.

```rust
#[rapx::requires(Allocated(ptr, u8, layout.size()))]
```

### `Typed(ptr, Ty)`

Memory holds a valid value of `Ty`. Does not check alignment тАФ use `Align` for that.

```rust
#[rapx::requires(Typed(ptr, T))]
```

### `ValidNum(predicate, ...)` / `ValidNum(value, "interval")`

Numeric constraints expressed as relational predicates or intervals.

```rust
#[rapx::requires(ValidNum(index < len))]
#[rapx::requires(ValidNum(size_of::<T>() * len <= isize::MAX))]
#[rapx::requires(ValidNum(mid, "[0, self.len]"))]
#[rapx::requires(ValidNum(N, "[1,)"))]
```

### `Alias(p1, p2)`

Two places refer to the same memory. Classified as a **hazard**: an unproved `Alias` does not block a `SOUND` verdict, but warns about potential aliasing issues.

```rust
#[rapx::requires(Alias(ptr, ret))]
```

### `Alive(ptr)`

The allocation is still live тАФ not freed or deallocated.

```rust
#[rapx::requires(Alive(ptr))]
```

### `ValidTransmute(Src, Dst)`

Reinterpreting memory from `Src` to `Dst` is valid тАФ `Dst` is structurally composed of `Src`.

```rust
#[rapx::requires(ValidTransmute(u32, [u8; 4]))]
```

### `TransmuteWithoutAlign([Src], [Dst])`

Every `size_of(Dst)`-byte window within `[Src]` is a valid `Dst` value without requiring alignment.

```rust
#[rapx::requires(TransmuteWithoutAlign([T], [U]))]
```

### `NonOverlap(a, b, ...)`

The specified memory ranges do not overlap.

```rust
#[rapx::requires(NonOverlap(src, dst))]
```

### `Deref(ptr)` / `Deref(ptr, count)` / `Deref(ptr, Ty, count)`

Composite of `Allocated` + `InBound`. The pointer can be safely dereferenced.

```rust
#[rapx::requires(Deref(ptr, count))]
```

### `Ptr2Ref` / `ValidPtr2Ref(ptr)`

A raw pointer can be safely converted to a reference.

```rust
#[rapx::requires(Ptr2Ref(ptr))]
```

### `ValidSlice(slice)` / `ValidSlice(slice, Ty)`

A slice reference is valid (non-null, aligned data, valid length).

```rust
#[rapx::ensures(ValidSlice(return, u8))]
```

### `ValidString(ptr)`

Byte data is valid UTF-8.

```rust
#[rapx::requires(ValidString(v))]
```

### `ValidCStr(ptr)`

C string is null-terminated with no interior null bytes.

```rust
#[rapx::requires(ValidCStr(ptr))]
```

### `Layout(layout)`

A `Layout` value matches a prior allocation's size and alignment.

### `Size`

Two forms: `Size(ptr)` asserts the type is not a ZST. `Size` alone (no args) asserts a generic `T` is `Sized` and non-zero.

### `NoPadding(ptr)`

The type has no padding bytes between fields.

### `Unwrap(ptr)`

The `Option`/`Result` is in the expected variant (`Some` / `Ok`).

### `Owning(ptr)`

The value has unique ownership тАФ no other write-capable alias exists.

```rust
#[rapx::invariant(Owning(ptr))]
```

### `Pinned(ptr)`

The target is pinned and its address will not change.

```rust
#[rapx::requires(Pinned(ptr))]
```

### `NonVolatile(ptr)`

Memory is not volatile or externally modified.

### `Opened(ptr)`

An OS resource (e.g. file descriptor) is valid and open.

### `Trait(ptr)`

Trait-object safety contract assertion (valid vtable).

### `Unreachable`

The code path is unreachable.

### `Null` / `Nullable(ptr)`

The pointer may be null тАФ safe for null-tolerant operations like `as_ref`. Used inside the `any(...)` combinator.

### `Unknown`

Placeholder for unrecognized tags; silently ignored during verification.

## 8.3.4 JSON Contracts

Standard-library functions that cannot be annotated upstream use JSON contracts in [`std-public-contracts.json`](https://github.com/safer-rust/RAPx/blob/main/rapx/src/verify/attribute/assets/std-public-contracts.json). Each entry maps a function path to a list of `{ "tag": ..., "args": [...] }` objects:

```json
{
  "core::ptr::const_ptr::add": [
    { "tag": "NonNull", "args": ["self"] },
    { "tag": "Align",   "args": ["self", "T"] },
    { "tag": "InBound", "args": ["self", "T", "count"] }
  ]
}
```

The argument strings use the same expression grammar as direct annotations. Path lookup first tries an exact match on the cleaned def-path, then falls back to wildcard segment replacement (e.g. `core::slice::<impl [T]>::*` тЖТ `core::slice::*`).
