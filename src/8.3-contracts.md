# Chapter 8.3. Safety Property Contracts

Safety property contracts are the machine-checkable safety obligations that RAPx verifies. They are defined in [`contract.rs`](https://github.com/safer-rust/RAPx/blob/main/rapx/src/verify/contract.rs) as the `PropertyKind` enum. A `Property` is a `PropertyKind` plus a vector of `PropertyArg` values.

There are **two surface syntaxes** that both parse into the same internal `Property` representation:

1. **Direct annotation** — `#[rapx::requires(...)]` / `#[rapx::invariant(...)]` attributes written in Rust source ([§8.3.2](#832-direct-annotation-syntax)).
2. **JSON contract database** — entries in `std-public-contracts.json` for standard-library functions that cannot be annotated in upstream code ([§8.3.4](#834-json-contracts)).

Both are parsed by `Property::new(tcx, def_id, tag, exprs)`. The direct-annotation path parses the attribute token stream into `syn::Expr` arguments; the JSON path parses each argument string into a `syn::Expr` (after token normalization) and calls the same constructor. This chapter documents the property catalog, the argument grammar shared by both surfaces, and then each surface in turn.

## 8.3.1 Property Kinds

The verifier recognizes a set of `PropertyKind` variants. Their semantics are defined in the [`primitive-sp.md`](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md) safety property specification. Full usage details for each kind are in [§8.3.3](#833-property-reference).

Not every recognized kind has a dedicated SMT checker. Kinds with an SMT backend are proved/disproved by Z3; the rest produce `Unknown` and do not contribute to a soundness verdict. See [Chapter 8.4.4](./8.4-pipeline.md#844-smt-check) for the module mapping.

## 8.3.2 Direct Annotation Syntax

Contracts written in source use `register_tool` tool attributes. The crate (or module) must enable the tool:

```rust
#![feature(register_tool)]
#![register_tool(rapx)]
```

### `#[rapx::requires(...)]`

Declares a precondition on an (usually `unsafe`) function. Each attribute holds one property; group with commas or repeat the attribute:

```rust
// Repeated attributes …
#[rapx::requires(ValidPtr(ptr, u32, 1))]
#[rapx::requires(Align(ptr, u32))]

// … are equivalent to a single grouped attribute
#[rapx::requires(ValidPtr(ptr, u32, 1), Align(ptr, u32))]
```

A complete example:

```rust
#[rapx::requires(ValidPtr(ptr, u32, len))]
#[rapx::requires(Align(ptr, u32))]
#[rapx::requires(InBound(ptr, u32, len))]
#[rapx::requires(ValidNum(index < len))]
pub unsafe fn write_slice(ptr: *mut u32, len: usize, index: usize) {
    // ...
}
```

### `#[rapx::verify]`

Marks a function as a verification entry point for `--mode targeted` ([Chapter 8.1.2](./8.1-overview.md#812-verification-modes)). It takes no arguments and is orthogonal to `#[rapx::requires]`.

### `#[rapx::invariant(...)]`

Declares a struct-level invariant that must hold for every instance at all observable points ([Chapter 8.4.6](./8.4-pipeline.md#846-struct-invariant-verification)):

```rust
#[rapx::invariant(ValidPtr(self.ptr, T, self.cap))]
#[rapx::invariant(Align(self.ptr, T))]
pub struct MyBuf<T> {
    ptr: *mut T,
    len: usize,
    cap: usize,
}
```

### `#[rapx::ensures(...)]`

Declares a postcondition on an **unsafe trait** method — a contract that every `impl` of the trait must satisfy. Whereas `#[rapx::requires]` defines obligations on the *caller*, `#[rapx::ensures]` defines obligations on the *implementor*:

```rust
#![feature(register_tool)]
#![register_tool(rapx)]

unsafe trait Buffer {
    #[rapx::ensures(ValidSlice(return, u8))]
    fn as_bytes(&self) -> &[u8];

    #[rapx::requires(InBound(index, u8, 1))]
    unsafe fn get_unchecked(&self, index: usize) -> u8;
}
```

- The special placeholder `return` refers to the method's return value.
- Postconditions on trait methods can reference `self` fields and method parameters, same as `requires`.
- `#[rapx::ensures]` **only** applies to `unsafe trait` methods. Regular functions do not use (or need) this attribute — their post-effects are inferred from the standard-library call summary database.
- Trait implementation verification is currently **deferred** (logged during `--prepare-targets` but not yet proven by the SMT engine).
- Trait methods that take `unsafe fn` should use `#[rapx::requires]` for their preconditions and may additionally use `#[rapx::ensures]` for postconditions visible to clients.

Contract properties inside `#[rapx::ensures(...)]` use the same `PropertyKind` catalog and argument grammar as `#[rapx::requires(...)]` ([§8.3.1](#831-property-kinds)).

### `any(...)` Combinator

The `any(...)` combinator represents logical OR between disjuncts, where commas inside each parenthesized disjunct mean logical AND:

```text
any((Null(p)), (P1(p, ...), P2(p, ...), ...))
```

The currently supported shape is a **null guard**: exactly two disjuncts, one being `Null(p)` alone, the other a conjunction of properties over the same place `p`. The semantics: the conjunct properties hold whenever `p` is non-null, and the whole contract is satisfied when `p` is null. This works in both `#[rapx::requires]` and `#[rapx::invariant]` positions:

- **In `requires`**: The SMT solver encodes `(p == 0) ∨ (P1 ∧ P2)`. If the caller's state satisfies either branch, the contract is proved. This is useful for functions like `ptr::as_ref()` that accept null pointers — the contract declares the pointer may be null, but must be valid and aligned when non-null:

```rust
#[rapx::requires(any(Null(self), (ValidPtr(self, T, 1), Align(self, T))))]
pub unsafe fn as_ref<'a, T>(self: *const T) -> Option<&'a T> { /* ... */ }
```

- **In `invariant`**: Used for struct fields that may be null (e.g., linked list tail pointers). Combined with dominance information from null checks in user code (`if ptr.is_null() { return; }`), the verifier determines which branch to prove:

```rust
#[rapx::invariant(any((Null(self.next)), (ValidPtr(self.next, Node, 1), Align(self.next, Node))))]
pub struct Node {
    value: u32,
    next: *mut Node,
}
```

In both cases, `any(Null(p), (P1, P2))` expands to individual `Property` objects for `P1` and `P2`, each tagged with the null-guard place. The SMT checker then asserts `p != 0` before proving each conjunct, which is equivalent to proving `(p == 0) ∨ P_i_holds`.

### Contract Kind Metadata

A property may carry a trailing `kind = "<string>"` tag inside the same attribute. Two `ContractKind` values are recognized:

- **`kind = "precond"`** (default): A normal safety precondition. An `Unproved` verdict for this kind contributes to the `UNSOUND` result.
- **`kind = "hazard"`**: An advisory warning about a known unsafety pattern. An `Unproved` verdict for this kind **does not** contribute to the `UNSOUND` count — it is printed as an informational `[hazard]` entry. Set via `kind = "hazard"` inside `#[rapx::requires]`:

```rust
#[rapx::requires(ValidNum(size_of::<T>() * len <= isize::MAX), kind = "hazard")]
pub unsafe fn from_raw_parts<T>(ptr: *const T, len: usize) -> *const [T] {
    // ...
}
```

The `Alias` property is always classified as hazard automatically. Hazard contracts are also used in `std-public-contracts.json` to flag known but currently unprovable constraints.

### Places accepted in annotations

- Parameter names: `self`, `ptr`, `index`, `len`.
- Field projections: `self.0`, `self.ptr`, `self.cap`.
- Length sugar: `self.len` is read as `len(self)`.
- Const generics: `N` resolves to the enclosing item's const parameter.
- Return value: `return` refers to the function's return value (only in `#[rapx::ensures]` context).

## 8.3.3 Property Reference

Each `PropertyKind` variant is documented below with its tag, argument syntax, semantics, and common usage patterns.

---

### `ValidPtr(ptr, Ty, count)`

**Arguments:** 1 Place + 1 Type + 1 Expr (exactly 3)

**Meaning:** `same_alloc([ptr, ptr + sizeof(Ty) * count])` — the memory range is within a single live allocation. This is the fundamental pointer-validity contract.

**Checks at SMT level:** decomposes into `Size(T,0) || Deref(p,T,n)`. ZSTs are trivially valid.

**Use cases:**
- `slice::from_raw_parts(ptr, len)` requires `ValidPtr(ptr, T, len)` 
- `ptr::read(ptr)` requires `ValidPtr(ptr, T, 1)`
- `ptr::write(ptr, val)` requires `ValidPtr(ptr, T, 1)`

---

### `Align(ptr, Ty)`

**Arguments:** 1 Place + 1 Type (exactly 2)

**Meaning:** `(ptr as usize) % align_of::<Ty>() == 0` — the pointer satisfies the alignment requirement of type `Ty`.

**Checks at SMT level:** lowered to `SmtObligation::Aligned { place, align }`. Proved via Z3 SMT solver; trivially true for 1-byte-aligned types or reference-derived pointers.

**Use cases:**
- Every raw pointer read/write requires `Align(ptr, T)`
- `slice::from_raw_parts(ptr, len)` requires `Align(ptr, T)`
- `ptr::add(ptr, count)` requires `Align(ptr, T)`

---

### `InBound` / `InBounded`

**Arguments:** 1, 2, or 3 args, depending on the form

**Meaning:** Every byte accessed through this pointer (or every element indexed in this slice) lies within the bounds of the owning allocation. This is the fundamental memory-safety property enforced for unchecked indexing and pointer arithmetic.

`InBound` accepts four surface shapes, each lowered to one of two SMT obligations:

| Surface shape | When accepted | SMT obligation |
|---|---|---|
| `InBound(index_access(slice, index))` | 1 arg, explicit `index_access` | `Predicate`: `0 <= lower <= upper <= len(slice)` |
| `InBound(slice, index)` | 2 args, first arg is a slice type | auto-expands to `InBound(index_access(slice, index))` (same as above) |
| `InBound(ptr, count)` | 2 args, first arg is a pointer (non-slice) | `InBounds`: `ptr` points to at least `count` elements within its object |
| `InBound(ptr, Ty, count)` | 3 args, explicit type | `InBounds`: as above, element size taken from `Ty` |

When the callee is a pointer-arithmetic primitive (e.g. `ptr::add`, `ptr::sub`), the checker upgrades the obligation to `PointerRangeInBounds`, which proves that **every** offset in the delta range `[lower_delta, upper_delta]` (from the base pointer `ptr`) stays inside the original allocation. For unsigned addition the range is `[0, count]`; for subtraction it is `[-count, 0]`; for signed byte-add primitives it is `[-count, count]`.

**The `index_access` form and `SliceIndex` types**

When `index_access(slice, index)` is used, the verifier inspects the **Rust type** of `index` to determine the lower and upper bounds to prove:

| Index type | Lower bound | Upper bound |
|---|---|---|
| `usize` (scalar) | `index` | `index + 1` |
| `Range<usize>` | `range.start` | `range.end` |
| `RangeFrom<usize>` | `range.start` | `len(slice)` |
| `RangeTo<usize>` | `0` | `range.end` |
| `RangeFull` | `0` | `len(slice)` |
| `RangeInclusive<usize>` | `range.start` | `range.end + 1` |
| `RangeToInclusive<usize>` | `0` | `range.end + 1` |

The verifier then asserts three conjuncts: `0 <= lower`, `lower <= upper`, and `upper <= len(slice)`.

**Pointer-arithmetic form**

For `InBound(ptr, Ty, count)` (or the two-arg shorthand where the type is inferred), the SMT model tracks pointer provenance through the forward simulation. It matches `ptr` back to the original allocation (from `alloc::alloc`, stack `alloca`, or a `Vec`/`Box` base pointer) and proves that `ptr + count * sizeof(Ty)` does not exceed the allocation end.

When the callee is `ptr::add`, `ptr::sub`, `ptr::offset`, or similar, the checker uses `PointerRangeInBounds` — it encodes that the **entire pointer-arithmetic delta range** is safe, catching off-by-one errors where only the final address was checked but an intermediate step overflowed.

**Use cases:**
- `<[T]>::get_unchecked(i)` requires `InBound(self, i)` or `InBound(index_access(self, i))`
- `ptr::add(ptr, count)` requires `InBound(ptr, T, count)`
- `ptr::sub(ptr, count)` requires `InBound(ptr, T, count)` — lowered to `PointerRangeInBounds` with `[-count, 0]`
- `<[T]>::get_disjoint_unchecked_mut(indices)` requires `InBound(self, indices)` (array-index checked by a trusted validator)

---

### `Init(ptr, Ty, count)`

**Arguments:** 1 Place + 1 Type + 1 Expr (exactly 3)

**Meaning:** `for all i in 0..count: *(ptr + i * sizeof(Ty)) holds TypeInvariant(Ty)` — memory is initialized for `count` elements of type `Ty`. The bytes have been written with valid values and are safe to read.

**Checks at SMT level:** lowered to `SmtObligation::Initialized`. Proved via KnownInit facts and SMT alias analysis.

**Use cases:**
- `slice::from_raw_parts(ptr, len)` requires `Init(ptr, T, len)`
- `MaybeUninit::assume_init()` requires `Init(self, T, 1)`
- `ptr::read(ptr)` requires `Init(ptr, T, 1)`

---

### `NonNull(ptr)`

**Arguments:** 1 Place (exactly 1)

**Meaning:** `ptr as usize != 0` — the pointer is not null.

**Checks at SMT level:** lowered to `SmtObligation::NonZero { place }`. Proved via Z3.

**Use cases:**
- `NonNull::new_unchecked(ptr)` requires `NonNull(ptr)`
- `slice::from_raw_parts(ptr, len)` requires `NonNull(ptr)`
- `ptr::add(ptr, count)` requires `NonNull(ptr)`

---

### `Allocated(ptr)` / `Allocated(ptr, Ty, count)`

**Arguments:** 1 Place, or 1 Place + 1 Type + 1 Expr

**Meaning:** `ptr points to live heap/stack allocation` — the memory belongs to a live allocation and the pointer is within its bounds.

**Checks at SMT level:** lowered to `SmtObligation::Allocated`.

**Use cases:**
- GlobalAlloc / Allocator methods: `Allocated(ptr, u8, layout.size())`
- Raw pointer arithmetic: `Allocated(ptr)`

---

### `Typed(ptr, Ty)`

**Arguments:** 1 Place + 1 Type (exactly 2)

**Meaning:** `*ptr holds TypeInvariant(Ty)` — the memory at `ptr` holds a valid value of type `Ty`. Uses path-local provenance tracking (cast chains, write facts, storage types) — does NOT check alignment (alignment is a separate `Align` contract).

**Checks at SMT level:** path-local proof via forward visitor fact tracing (not numeric SMT). Traces casts, writes, references, and storage types.

**Use cases:**
- `align_to::<U>()`: `Typed(self, U)` — the slice data can be reinterpreted as `U` elements
- `ptr::read(ptr)`: requires `Typed(ptr, T)` — data forms a valid `T`
- `*raw_ptr` (read): requires `Typed(ptr, T)`

---

### `ValidNum(predicate, ...)` / `ValidNum(value, "interval")`

**Arguments:** variadic — one or more predicates, or a value + quoted interval string

**Meaning:** Numeric constraints expressed as relational predicates or interval bounds.

**Checks at SMT level:** lowered to `SmtObligation::Predicate`. Proved via Z3.

**Common patterns:**
- Slice validity: `ValidNum(size_of::<T>() * len <= isize::MAX)`
- Index bounds: `ValidNum(index < len)`, `ValidNum(mid, "[0, self.len]")`
- Const-generic guards: `ValidNum(N)`, `ValidNum(N, "[1,)")`
- Divisibility: `ValidNum(self.len() % N == 0)`

---

### `Alias(p1, p2)`

**Arguments:** 2 Places (exactly 2)

**Meaning:** `p1` and `p2` alias each other — they refer to the same memory.  Unlike most property kinds, `Alias` is a **hazard** (contract kind `Hazard`): it does not block a `SOUND` verdict when unproved.  The checker treats it as a post-call alias-conflict check: after the callee creates a view (e.g. `from_raw_parts_mut` returns a `&mut [T]`), writing or reading through the original raw pointer `p1` while the view `p2` is live constitutes a hazard violation.

**Checks at SMT level:** proven via path-sensitive forward scan for conflicting raw accesses and structure-level alias analysis.

**Use cases:**
- `slice::from_raw_parts(ptr, len)` requires `Alias(ptr, ret)` — creating a shared slice view creates an alias between the source pointer and the returned slice
- `slice::from_raw_parts_mut(ptr, len)` requires `Alias(ptr, ret)` — creating a mutable slice creates an alias hazard
- `ptr::copy(src, dst, count)` requires `Alias(src, dst)` — the two pointers may overlap
- `Box::from_raw(ptr)` requires `Alias(ptr, ret)` — reconstructing an owner from a raw pointer

---

### `Alive(ptr)`

**Arguments:** 1 Place (exactly 1)

**Meaning:** `lifetime(*ptr) within lifetimes(caller_args)` — the allocation that `ptr` points into is still live (not freed/deallocated).

**Checks at SMT level:** proven via liveness analysis.

**Use cases:**
- `slice::from_raw_parts(ptr, len)` requires `Alive(ptr)`

---

### `ValidTransmute(Src, Dst)`

**Arguments:** 2 Types (exactly 2)

**Meaning:** `bytes_of(Dst) within bytes_of(Src)` — reinterpreting a region of `Src` values as `Dst` is valid; every bit pattern produced by valid `Src` values is a valid `Dst`.

**Structural check (`composed_entirely_of`):**
- Exact type equality
- Arrays: `[Src; N]` composed of `Src`
- Tuples: `(Src, Src, ...)` composed of `Src`
- `repr(simd)` structs: `Simd<T, N>` composed of `T`
- `repr(transparent)` structs: newtype over `Src`
- Recursive depth limit of 16

**Use cases:**
- `transmute::<Src, Dst>(src)`: `ValidTransmute(Src, Dst)`
- `slice::align_to::<U>()`: `ValidTransmute(T, U)` (std lib source)

---

### `TransmuteWithoutAlign([Src], [Dst])`

**Arguments:** 2 Types (exactly 2, written as slice notation `[T]`/`[U]`)

**Meaning:** Every `size_of(Dst)`-byte contiguous window within the byte-stream of `[Src]`, when reinterpreted as `Dst`, yields a value satisfying `type_invariant(Dst)` without requiring alignment of `Dst`. Formally:

\[
\forall w \subset \text{bytes}([Src]),\; |w| = |Dst|: \text{reinterpret\_as}_{Dst}(w) \models \text{type\_invariant}(Dst) \setminus \text{align\_of}(Dst)
\]

This covers all size ratios: `|U| > |T|` (multiple T per U, with or without remainder), `|U| < |T|` (T split into multiple U), and boundary-agnostic sliding windows.

**Check proceeds in 4 tiers:**
1. **Structural** — `composed_entirely_of` (same as `ValidTransmute`) for arrays, SIMD, tuples, transparent wrappers
2. **SIMD compat** — accepts any `repr(simd)` type as compatible
3. **All-bit-patterns-valid** — destination type has no validity constraints beyond its field types (e.g. `u8`, `u16`, `u32`, `f64`, raw pointers, and structs/tuples thereof without constrained scalar ranges like `NonZero` or `bool`). Proved via `all_bit_patterns_valid()` which checks field types recursively and inspects `BackendRepr::Scalar::valid_range` for non-trivial constraints.
4. **Size-aware** — when sizes are known but none of the above succeed, returns `Unknown` (cannot prove `type_invariant` for every window). Size divisibility is *not* a deciding factor; the former `proved` fallback for divisible sizes has been demoted to `Unknown`.

**Use cases:**
- `slice::align_to::<U>()`: the JSON contract in `std-public-contracts.json` declares `TransmuteWithoutAlign([T], [U])` for both `align_to` and `align_to_mut`. Callers must either add `#[rapx::requires(TransmuteWithoutAlign([T], [U]))]` on their function or rely on the verifier to prove it at the call site (e.g. when `Dst` is a type like `u8`/`u32` that passes `all_bit_patterns_valid`).

---

### `NonOverlap(a, b, ...)`

**Arguments:** variadic — 2 or more Places

**Meaning:** `[args] are pairwise disjoint memory ranges` — the specified memory ranges do not overlap. For two pointers, `[a, a+size) ∩ [b, b+size) = ∅`.

**Checks at SMT level:** lowered to `SmtObligation::NonOverlapping`.

**Use cases:**
- `ptr::copy_nonoverlapping(src, dst, count)` requires `NonOverlap(src, dst)`
- `ptr::swap_nonoverlapping(a, b, count)` requires `NonOverlap(a, b)`
- `<[T]>::get_disjoint_unchecked_mut(indices)` requires `NonOverlap(indices)`

---

### `Deref(ptr)` / `Deref(ptr, count)` / `Deref(ptr, Ty, count)`

**Arguments:** 1 Place, or 1 Place + 1 Expr, or 1 Place + 1 Type + 1 Expr

**Meaning:** `same_alloc(ptr) and 0 <= byte_offset(ptr) < alloc_len(ptr)` — composite of Allocated + InBound. The pointer can be safely dereferenced.

**Use cases:**
- Raw pointer dereference: `Deref(ptr)` or `Deref(ptr, count)`

---

### `ValidPtr2Ref` / `Ptr2Ref(ptr)`

**Arguments:** 1 Place (exactly 1)

**Meaning:** `can soundly convert ptr to &/&mut reference` — a raw pointer meets all reference safety requirements.

**Use cases:**
- Converting a raw pointer to a reference (pointer-to-reference cast)

---

### `ValidSlice(slice)` / `ValidSlice(slice, Ty)`

**Arguments:** 1 Place, or 1 Place + 1 Type

**Meaning:** `slice points to valid slice with non-null aligned data` — the pointer is valid for the declared length.

**Use cases:**
- Validating slice-based data structures

---

### `ValidString(ptr)`

**Arguments:** 1 Place (exactly 1)

**Meaning:** `v is valid UTF-8` — the byte data at `ptr` is valid UTF-8.

**Use cases:**
- `str::from_utf8_unchecked(v)`: `ValidString(v)`

---

### `ValidCStr(ptr)`

**Arguments:** 1 Place (exactly 1)

**Meaning:** `ptr is a null-terminated valid UTF-8 byte sequence` — the C string at `ptr` has no interior null bytes.

**Use cases:**
- `CStr::from_ptr(ptr)`: `ValidCStr(ptr)` + `ValidPtr(ptr, c_char, 1)` + `NonNull(ptr)`

---

### `Layout(layout)`

**Arguments:** 1 Place (exactly 1)

**Meaning:** `layout matches prior allocation size and alignment`.

**Use cases:**
- Allocator methods: `Layout(layout)`

---

### `Size`

**Arguments:** 1 Place, or none

**Meaning:** Two forms:
- `Size(ptr)` (with argument): `sizeof(type_pointed_by(ptr)) > 0` — the type is not a ZST
- `Size` (no arguments): `Size(T, != 0)` — the type parameter `T` is `Sized` and non-zero. Required for generic pointer offset arithmetic (`ptr::add`, `ptr::sub`, etc.)

---

### `NoPadding(ptr)`

**Arguments:** 1 Place (exactly 1)

**Meaning:** `T has no padding bytes between fields`.

---

### `Unwrap(ptr)`

**Arguments:** 1 Place (exactly 1)

**Meaning:** `x is in expected variant / not None / not Err`.

---

### `Owning(ptr)`

**Arguments:** 1 Place (exactly 1)

**Meaning:** `ptr has unique ownership, no other write-capable alias`.

---

### `Pinned(ptr)`

**Arguments:** 1 Place (exactly 1)

**Meaning:** `ptr will not be moved` — the target's address will not change.

**Use cases:**
- `Pin::new_unchecked(ptr)`: `Pinned(ptr)`

---

### `NonVolatile(ptr)`

**Arguments:** 1 Place (exactly 1)

**Meaning:** `ptr does not reference volatile memory`.

---

### `Opened(ptr)`

**Arguments:** 1 Place (exactly 1)

**Meaning:** `fd is a valid open file descriptor`.

**Use cases:**
- `from_raw_fd(fd)`: `Opened(fd)`

---

### `Trait(ptr)`

**Arguments:** 1 Place (exactly 1)

**Meaning:** `T upholds its unsafe trait contract` — trait-object safety contract, vtable is valid.

---

### `Unreachable`

**Arguments:** none

**Meaning:** `not Reachable()` — the code path is unreachable. Used for `hint::unreachable_unchecked()`.

---

### `Null` → `Nullable(ptr)`

**Arguments:** 1 Place (exactly 1)

**Meaning:** `ptr may be null` — safe for null-tolerant operations like `as_ref`.

### `Unknown`

**Arguments:** any

**Meaning:** Any tag not recognized by the parser; ignored during checking. Reserved for future contract types.

## 8.3.4 JSON Contracts

Standard-library functions that cannot be annotated upstream use JSON contracts in [`std-public-contracts.json`](https://github.com/safer-rust/RAPx/blob/main/rapx/src/verify/attribute/assets/std-public-contracts.json). Each entry maps a function path to a list of `{ "tag": ..., "args": [...] }` objects:

```json
{
  "core::ptr::const_ptr::add": [
    { "tag": "NonNull", "args": ["self"] },
    { "tag": "Align",   "args": ["self", "T"] },
    { "tag": "InBound", "args": ["self", "T", "count"] }
  ]
}
```

The argument strings use the same expression grammar as direct annotations. Path lookup first tries an exact match on the cleaned def-path, then falls back to wildcard segment replacement (e.g. `core::slice::<impl [T]>::*` → `core::slice::*`).
