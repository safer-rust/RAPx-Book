# Chapter 8.3. Safety Property Contracts

RAPx verifies safety through explicit contracts declared in source code. This chapter covers how to write contracts using the available property kinds, and how to inspect resolved contracts with `--debug-contracts`.

Contracts are declared in two ways:

1. Direct annotation — `#[rapx::requires(...)]`, `#[rapx::invariant(...)]` attributes in Rust source ([§8.3.2](#832-direct-annotation-syntax)).
2. JSON database — pre-written contracts for standard-library functions that cannot be annotated upstream ([§8.3.4](#834-json-contracts)).

## 8.3.1 Property Kinds

The semantics of each property kind are defined in [`primitive-sp.md`](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md). An annotated reference with examples for each kind is in [§8.3.3](#833-property-reference).

## 8.3.2 Direct Annotation Syntax

Contracts written in source use `register_tool` tool attributes. The crate (or module) must enable the tool:

```rust
#![feature(register_tool)]
#![register_tool(rapx)]
```

### `#[rapx::requires(...)]`

Declares a precondition on an (usually `unsafe`) function. Each attribute holds one property; group with commas or repeat the attribute:

```rust
// Repeated attributes …
#[rapx::requires(ValidPtr(ptr, u32, 1))]
#[rapx::requires(Align(ptr, u32))]

// … are equivalent to a single grouped attribute
#[rapx::requires(ValidPtr(ptr, u32, 1), Align(ptr, u32))]
```

A complete example:

```rust
#[rapx::requires(ValidPtr(ptr, u32, len))]
#[rapx::requires(Align(ptr, u32))]
#[rapx::requires(InBound(ptr, u32, len))]
#[rapx::requires(ValidNum(index < len))]
pub unsafe fn write_slice(ptr: *mut u32, len: usize, index: usize) {
    // ...
}
```

### `#[rapx::verify]`

Marks a function as a verification entry point for `--mode targeted` ([Chapter 8.1.2](./8.1-overview.md#812-verification-modes)). It takes no arguments and is orthogonal to `#[rapx::requires]`.

### `#[rapx::invariant(...)]`

Declares a struct-level invariant that must hold for every instance at all observable points ([Chapter 8.4.6](./8.4-pipeline.md#846-struct-invariant-verification)):

```rust
#[rapx::invariant(ValidPtr(ptr, T, cap))]
#[rapx::invariant(Align(ptr, T))]
pub struct MyBuf<T> {
    ptr: *mut T,
    len: usize,
    cap: usize,
}
```

### `any(...)` Combinator

The `any(...)` combinator represents logical OR between disjuncts, where commas inside each parenthesized disjunct mean logical AND:

```text
any((Null(p)), (P1(p, ...), P2(p, ...), ...))
```

The currently supported shape is a null guard: exactly two disjuncts, one being `Null(p)` alone, the other a conjunction of properties over the same place `p`. The semantics: the conjunct properties hold whenever `p` is non-null, and the whole contract is satisfied when `p` is null. This works in both `#[rapx::requires]` and `#[rapx::invariant]` positions:

- In `requires`: The SMT solver encodes `(p == 0) ∨ (P1 ∧ P2)`. If the caller's state satisfies either branch, the contract is proved. This is useful for functions like `ptr::as_ref()` that accept null pointers — the contract declares the pointer may be null, but must be valid and aligned when non-null:

```rust
#[rapx::requires(any(Null(self), (ValidPtr(self, T, 1), Align(self, T))))]
pub unsafe fn as_ref<'a, T>(self: *const T) -> Option<&'a T> { /* ... */ }
```

- In `invariant`: Used for struct fields that may be null (e.g., linked list tail pointers). Combined with dominance information from null checks in user code (`if ptr.is_null() { return; }`), the verifier determines which branch to prove:

```rust
#[rapx::invariant(any((Null(next)), (ValidPtr(next, Node, 1), Align(next, Node))))]
pub struct Node {
    value: u32,
    next: *mut Node,
}
```

In both cases, `any(Null(p), (P1, P2))` expands to individual `Property` objects for `P1` and `P2`, each tagged with the null-guard place. The SMT checker then asserts `p != 0` before proving each conjunct, which is equivalent to proving `(p == 0) ∨ P_i_holds`.

### Contract Kind Metadata

A property may carry a trailing `kind = "<string>"` tag inside the same attribute. Two `ContractKind` values are recognized:

- `kind = "precond"` (default): A normal safety precondition. An `Unproved` verdict for this kind contributes to the `UNSOUND` result.
- `kind = "hazard"`: An advisory warning about a known unsafety pattern. An `Unproved` verdict for this kind does not contribute to the `UNSOUND` count — it is printed as an informational `[hazard]` entry. Set via `kind = "hazard"` inside `#[rapx::requires]`:

```rust
#[rapx::requires(ValidNum(size_of::<T>() * len <= isize::MAX), kind = "hazard")]
pub unsafe fn from_raw_parts<T>(ptr: *const T, len: usize) -> *const [T] {
    // ...
}
```

The `Alias` property is always classified as hazard automatically. Hazard contracts are also used in `std-public-contracts.json` to flag known but currently unprovable constraints.

### Places accepted in annotations

- Parameter names in `requires`: `self`, `ptr`, `index`, `len`.
- Field names in `invariant`: `ptr`, `cap`, `head` — no `self.` prefix.
- Field projections in `requires` (via `self`): `self.0`, `self.ptr`.
- Length sugar: `self.len` is read as `len(self)`.
- Const generics: `N` resolves to the enclosing item's const parameter.

## 8.3.3 Property Reference

### `ValidPtr(ptr, Ty, count)`

`(ptr as usize) % align_of::<Ty>() != 0` — the pointer is valid for reading and writing `count` elements of `Ty`. Defined in [primitive-sp §2.2](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#22-compound-sps-used-in-rustdoc): for ZSTs vacuously true; for non-ZSTs equivalent to `Allocated ∧ InBound`.

```rust
#[rapx::requires(ValidPtr(ptr, u32, len))]
```

### `Align(ptr, Ty)`

`ptr % alignment(Ty) == 0` — the pointer satisfies the alignment requirement of `Ty`. [psp I.1](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#311-alignment).

```rust
#[rapx::requires(Align(ptr, u32))]
```

### `InBound` / `InBounded`

The accessed memory lies within the bounds of a single allocated object. Two forms:

| Form | Example |
|------|---------|
| `InBound(slice, index)` | `#[rapx::requires(InBound(self, i))]` |
| `InBound(ptr, Ty, count)` | `#[rapx::requires(InBound(ptr, u32, len))]` |

The slice form supports all `SliceIndex` types and expands to the correct bounds check. [psp II.3](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#321-allocation).

### `Init(ptr, Ty, count)`

Memory is initialized for `count` elements of `Ty`. Stronger than `Typed`: Init implies Typed but not vice versa. [psp III.4](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#333-initialization).

```rust
#[rapx::requires(Init(ptr, T, len))]
```

### `NonNull(ptr)`

`ptr != 0` — the pointer is not null. [psp II.1](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#address).

```rust
#[rapx::requires(NonNull(ptr))]
```

### `Allocated(ptr, Ty, count, [allocator])`

Memory belongs to a live allocation. The fourth argument (allocator) defaults to `global` when omitted. [psp II.2](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#321-allocation).

```rust
#[rapx::requires(Allocated(ptr, u8, layout.size()))]
#[rapx::requires(Allocated(ptr, u8, layout.size(), Global))]
```

### `Typed(ptr, Ty)`

`*p` satisfies `TypeInvariant(T)` — the memory was created as type `T` and has not been type-punned. Weaker than `Init`: does not require initialized content. [psp III.6](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#335-typed).

```rust
#[rapx::requires(Typed(ptr, T))]
```

### `ValidNum(expr, vrange)` / `ValidNum(predicate, ...)`

Numeric constraints: a predicate expression or an interval range. [psp III.1](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#331-integer).

```rust
#[rapx::requires(ValidNum(index < len))]
#[rapx::requires(ValidNum(size_of::<T>() * len <= isize::MAX))]
#[rapx::requires(ValidNum(mid, "[0, self.len]"))]
```

### `Owning(ptr)`

`ownership(*p) == none` — the pointer is the sole carrier of ownership; no live owner aliases the pointee. [psp IV.1](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#341-ownership).

```rust
#[rapx::invariant(Owning(ptr))]
```

### `Alias(p1, p2)`

`p1 == p2` — the two places refer to the same memory. Classified as a hazard: an unproved `Alias` does not block `SOUND`. [psp IV.2](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#342-alias).

```rust
#[rapx::requires(Alias(ptr, ret))]
```

### `Alive(ptr, 'a)`

`lifetime(*p) >= l` — the allocation is still live and not freed across the lifetime `'a`. [psp IV.3](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#343-lifetime).

```rust
#[rapx::requires(Alive(ptr, 'a))]
```

### `NonOverlap(a, b, ...)`

The specified memory ranges are pairwise disjoint. Two or more places. [psp II.4](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#321-allocation).

```rust
#[rapx::requires(NonOverlap(src, dst))]
```

### `Deref(ptr)` / `Deref(ptr, count)` / `Deref(ptr, Ty, count)`

`Allocated ∧ InBound` — the pointer can be safely dereferenced. [primitive-sp §2.2](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#22-compound-sps-used-in-rustdoc).

```rust
#[rapx::requires(Deref(ptr, count))]
```

### `Ptr2Ref` / `ValidPtr2Ref(ptr)`

`Init(p, T, 1) ∧ Align(p, T) ∧ Alias(p, ret)` — a raw pointer meets all requirements for sound reference conversion. [primitive-sp §2.2](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#22-compound-sps-used-in-rustdoc).

```rust
#[rapx::requires(Ptr2Ref(ptr))]
```

### `ValidSlice(slice)` / `ValidSlice(slice, Ty)`

A slice reference is valid — non-null, aligned data, valid length.

```rust
#[rapx::requires(ValidSlice(slice, u8))]
```

### `ValidString(ptr)`

The byte data at `ptr` is valid UTF-8. [psp III.2](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#332-string).

```rust
#[rapx::requires(ValidString(v))]
```

### `ValidCStr(ptr)`

The C string at `ptr` is null-terminated with no interior null bytes. [psp III.3](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#332-string).

```rust
#[rapx::requires(ValidCStr(ptr))]
```

### `Layout(layout)`

A `Layout` value matches a prior allocation's size and alignment. [primitive-sp §2.2](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#22-compound-sps-used-in-rustdoc).

```rust
#[rapx::requires(Layout(layout))]
```

### `Size`

`Size(T, c)` — two forms: `Size(ptr)` asserts the pointee type is non-ZST; `Size` alone asserts a generic `T` is `Sized` and non-zero. [psp I.2](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#312-size).

```rust
#[rapx::requires(Size)]
```

### `NoPadding(ptr)`

`padding(T) == 0` — the type has no padding bytes. [psp I.3](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#313-padding).

```rust
#[rapx::requires(NoPadding(ptr))]
```

### `Unwrap(ptr)`

The `Option`/`Result` is in the expected variant (`Some` / `Ok`). [psp III.5](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#334-unwrap).

```rust
#[rapx::requires(Unwrap(ptr))]
```

### `Pinned(ptr)`

The target is pinned — its address will not change. [psp V.1](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#351-pin).

```rust
#[rapx::requires(Pinned(ptr))]
```

### `NonVolatile(ptr)`

Memory is not volatile — no other thread writes to the region. [psp V.2](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#352-volatility).

```rust
#[rapx::requires(NonVolatile(ptr))]
```

### `Opened(fd)`

An OS resource (e.g. file descriptor) is valid and open. [psp V.3](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#353-opened-file).

```rust
#[rapx::requires(Opened(fd))]
```

### `Trait(T, trait)`

The type `T` implements the specified trait — `trait ∈ traitimpl(T)`. [psp V.4](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#354-trait).

```rust
#[rapx::requires(Trait(T, Copy))]
```

### `Unreachable`

The code path is unreachable. [psp V.5](https://github.com/safer-rust/safety-tags/blob/main/primitive-sp.md#355-unreachable).

```rust
#[rapx::requires(Unreachable)]
```

### `ValidTransmute(Src, Dst)`

Reinterpreting memory from `Src` to `Dst` is valid — `Dst` is structurally composed of `Src` (exact type equality, array/tuple/simd/transparent decomposition).

```rust
#[rapx::requires(ValidTransmute(u32, [u8; 4]))]
```

### `TransmuteWithoutAlign([Src], [Dst])`

Every `size_of(Dst)`-byte window within `[Src]` is a valid `Dst` value, without requiring alignment. Checked via structural match, SIMD compat, or all-bit-patterns-valid analysis.

```rust
#[rapx::requires(TransmuteWithoutAlign([T], [U]))]
```

### `Null` / `Nullable(ptr)`

The pointer may be null — used inside the `any(...)` combinator for null-guarded contracts.

```rust
#[rapx::requires(any(Null(self), (ValidPtr(self, T, 1), Align(self, T))))]
```

### `Unknown`

Placeholder for unrecognized tags; silently ignored during verification.

## 8.3.4 JSON Contracts

Standard-library functions that cannot be annotated upstream use JSON contracts in [`std-public-contracts.json`](https://github.com/safer-rust/RAPx/blob/main/rapx/src/verify/attribute/assets/std-public-contracts.json). Each entry maps a function path to a list of `{ "tag": ..., "args": [...] }` objects:

```json
{
  "core::ptr::const_ptr::add": [
    { "tag": "NonNull", "args": ["self"] },
    { "tag": "Align",   "args": ["self", "T"] },
    { "tag": "InBound", "args": ["self", "T", "count"] }
  ]
}
```

The argument strings use the same expression grammar as direct annotations. Path lookup first tries an exact match on the cleaned def-path, then falls back to wildcard segment replacement (e.g. `core::slice::<impl [T]>::*` → `core::slice::*`).
